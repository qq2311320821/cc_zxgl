# 专项管理系统 PRD

## 1. 产品概述

### 1.1 产品背景

随着专项项目数量逐年增加，项目信息、专项文档、方案管理缺乏统一平台，导致：
- 项目信息分散，查找困难
- 专项建设方案无法复用
- 重复性工作多，效率低下

### 1.2 产品定位

面向项目管理人员的企业级专项管理平台，聚焦专项及项目的全生命周期管理，并提供基于AI的建设方案生成能力。

### 1.3 目标用户

- 项目经理
- 专项负责人
- 管理人员

## 3. 功能模块详述

### 3.1 登录模块

| 功能点 | 说明 |
|--------|------|
| 用户登录 | 用户名 + 密码登录 |
| 记住密码 | 勾选后记住登录状态 |
| 退出登录 | 清除会话，返回登录页 |

> 暂不设权限控制，所有登录用户可访问全部功能

---

### 3.2 专项管理

#### 3.2.1 专项列表

- 展示所有专项，支持按名称/编号搜索
- 显示字段：专项编号、专项名称、专项负责人、交付状态、需求模板、Prompt模板、描述图片、创建时间
- **支持横向滚动**：表格列过多时可左右滚动查看
- **操作按钮**：
  - 新增专项
  - 新增Prompt模板
  - 上传需求模板

#### 3.2.2 专项详情

| 字段 | 类型 | 说明 |
|------|------|------|
| 专项编号 | 文本 | 必填，唯一 |
| 专项名称 | 文本 | 必填 |
| 专项负责人 | 文本 | 必填 |
| 交付状态 | 下拉选择 | 必填，选项：0-1（产研主导）、1-3（产研交付专家共同交付）、3-100（交付专家赋能项目） |
| 需求模板 | 文件 | .xlsx格式，上传/下载，用于统一专项的需求收集格式 |
| Prompt模板 | 下拉选择 | 可选，关联1对1的Prompt模板 |
| 专项描述 | 图片 | 必填，上传1张图片 |
| 创建时间 | 自动 | 系统生成 |

#### 3.2.3 专项操作

- **新增专项**：填写专项信息，上传描述图片，可选择关联Prompt模板
- **编辑**：修改专项信息，支持重新上传图片
- **新增Prompt模板**：（独立操作入口）
  - 填写模板名称
  - 选择关联专项（1对1）
  - 填写模板内容
- **上传需求模板**：（独立操作入口）
  - 选择目标专项
  - 上传Excel需求模板文件
- **下载需求模板**：下载专项关联的Excel模板，供项目填写
- **删除**：删除专项（关联项目需处理）

---

### 3.3 项目管理

#### 3.3.1 项目列表

- 展示所有项目，支持按项目名称、客户名称搜索
- 显示字段：项目名称、项目编号、客户名称、客户编号、项目经理、所属专项、建设方案、需求文档、创建时间
- **支持横向滚动**：表格列过多时可左右滚动查看
- 支持新增项目

#### 3.3.2 项目详情

| 字段 | 类型 | 说明 |
|------|------|------|
| 项目名称 | 文本 | 必填 |
| 项目编号 | 文本 | 必填，唯一 |
| 客户名称 | 文本 | 必填 |
| 客户编号 | 文本 | 必填 |
| 项目经理 | 文本 | 必填 |
| 所属专项 | 下拉选择 | 必填，关联已存在的专项 |
| 专项建设方案 | 文件 | .docx格式，上传覆盖 |
| 需求文档 | 文件 | .xlsx格式，1对1，上传覆盖 |
| 创建时间 | 自动 | 系统生成 |

#### 3.3.3 项目操作

- **新增**：填写项目基本信息，选择所属专项
- **编辑**：修改项目信息，可重新上传专项建设方案、需求文档
- **上传/替换建设方案**：上传.docx文件，直接覆盖历史版本
- **上传/替换需求文档**：上传.xlsx文件，直接覆盖历史版本（使用专项关联的需求模板格式）
- **下载需求模板**：项目可下载所属专项的需求模板Excel，填写后上传
- **删除**：删除项目

---

### 3.4 AI方案生成

#### 3.4.1 Prompt模板管理（专项子项）

- Prompt模板与专项为1对1关系
- 支持创建/编辑AI提示词模板
- 模板可引用专项名称、负责人等变量
- **重要变更**：Prompt模板管理入口已移至专项管理页面，不再作为独立菜单

#### 3.4.2 需求模板管理（专项子项）

- 需求模板作为专项的Excel文件直接上传（1对1）
- 系统提供标准的Excel需求收集模板，包含以下字段：

| 字段名 | 说明 | 示例 |
|--------|------|------|
| 项目名称 | 项目名称 | XX系统建设项目 |
| 项目类型 | 项目类型 | 新建/扩建/改建 |
| 建设目标 | 项目建设要达到的目标 | 实现XX功能 |
| 核心功能 | 核心功能需求 | 用户管理、权限控制 |
| 性能要求 | 性能相关要求 | 支持1000并发 |
| 安全要求 | 安全相关要求 | 等保二级 |
| 集成需求 | 与其他系统的集成需求 | 与XX系统对接 |
| 预算范围 | 项目预算范围 | 100-200万 |
| 实施周期 | 预计实施周期 | 6个月 |
| 其他需求 | 其他补充说明 | 无 |

- 专项上传需求模板后，旗下项目可下载该模板，填报后重新上传作为项目需求文档

#### 3.4.3 选择专项与项目

- 选择目标专项
- 选择目标项目（需选择同一专项下的项目）
- 系统自动获取专项关联的Prompt模板、项目上传的需求文档内容

#### 3.4.4 AI生成

- **前置条件**：项目必须已上传建设方案和需求文档，方可生成
- **输入**：
  - 项目上传的需求文档内容（.xlsx）
  - 专项建设方案文档内容
  - Prompt模板（如已配置）
- **处理**：调用AI模型，结合模板与建设方案，生成新项目专属建设方案
- **输出**：
  - 页面展示AI生成的建设方案内容
  - 支持导出为Word格式（.docx）

#### 3.4.5 方案管理

- 可多次生成，保留历史生成记录
- 支持复制、编辑生成内容
- 导出Word功能

## 4. 数据模型

### 4.1 专项表 (special_project)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 主键 |
| code | VARCHAR(50) | 是 | 专项编号（唯一） |
| name | VARCHAR(100) | 是 | 专项名称 |
| leader | VARCHAR(50) | 是 | 专项负责人 |
| status | INT | 是 | 交付状态：0=产研主导，1=共同交付，2=专家赋能 |
| requirement_template_url | VARCHAR(500) | 否 | 需求模板Excel文件URL |
| prompt_id | UUID | 否 | 关联的Prompt模板ID（1对1） |
| description_image | VARCHAR(500) | 是 | 描述图片URL |
| created_at | DATETIME | 是 | 创建时间 |
| updated_at | DATETIME | 是 | 更新时间 |

### 4.2 项目表 (project)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 主键 |
| name | VARCHAR(100) | 是 | 项目名称 |
| code | VARCHAR(50) | 是 | 项目编号（唯一） |
| customer_name | VARCHAR(100) | 是 | 客户名称 |
| customer_code | VARCHAR(50) | 是 | 客户编号 |
| project_manager | VARCHAR(50) | 是 | 项目经理 |
| special_project_id | UUID | 是 | 所属专项ID |
| construction_plan_url | VARCHAR(500) | 是 | 建设方案文件URL |
| requirement_doc_url | VARCHAR(500) | 是 | 需求文档文件URL（1对1） |
| created_at | DATETIME | 是 | 创建时间 |
| updated_at | DATETIME | 是 | 更新时间 |

### 4.3 Prompt模板表 (prompt_template)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 主键 |
| name | VARCHAR(100) | 是 | 模板名称 |
| special_id | UUID | 是 | 关联的专项ID（1对1） |
| content | TEXT | 是 | Prompt模板内容 |
| created_at | DATETIME | 是 | 创建时间 |

### 4.4 AI生成方案表 (ai_plan)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 主键 |
| project_id | UUID | 是 | 关联项目ID |
| content | TEXT | 是 | AI生成的内容 |
| created_at | DATETIME | 是 | 生成时间 |

## 5. 页面结构

```
├── 登录页
│
├── 专项管理（包含Prompt模板和需求模板操作入口）
│   ├── 专项列表页（支持横向滚动）
│   │   ├── 新增专项按钮
│   │   ├── 新增Prompt模板按钮
│   │   └── 上传需求模板按钮
│   └── 专项新增/编辑弹窗
│
├── 项目管理
│   ├── 项目列表页（支持横向滚动）
│   └── 项目新增/编辑弹窗
│
└── AI方案生成
    ├── 选择专项/项目面板
    ├── AI生成结果面板
    └── 导出功能
```

> 注：Prompt模板管理不再作为独立页面，入口已整合至专项管理页面

## 6. 非功能性需求

| 需求项 | 说明 |
|--------|------|
| 文件大小限制 | 单个文件不超过20MB |
| 文件格式 | 仅支持.docx和.xlsx |
| 响应时间 | 页面操作响应时间 < 2秒 |
| 数据存储 | 云端存储（OSS/数据库） |
| 兼容性 | 支持主流浏览器（Chrome、Edge、Safari） |
